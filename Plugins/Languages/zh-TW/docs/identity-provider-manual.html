身分提供者手冊

簡介

透過使用身分提供者，您可以透過LDAP、Active Directory、OAuth2/OpenID Connect等 外部目錄服務進行使用者身分驗證。

支援的提供者類型：

  • LDAP - 標準輕量級目錄存取協定
  • Active Directory - Microsoft Active Directory
  • OAuth2 - OAuth 2.0 授權框架
  • OpenID Connect - 基於OAuth 2.0的身分層

客戶匯入

啟用「用於客戶匯入」後，將匯入以下資料：

  • 姓名 - sn（姓氏）或 displayName
  • 名字 - givenName
  • 職位 - title（職位名稱）
  • 公司 - company
  • 地址 - street/streetAddress、postalCode、l（城市）、st（省/州）、c（國家）
  • 性別 - Active Directory情況下，從personalTitle（Herr/Frau）取得

注意：使用OpenLDAP時，性別預設為「中性」，可手動修正。

提示

使用Synology NAS作為LDAP伺服器

您可以使用Synology NAS作為LDAP伺服器。在這種情況下，需要安裝「LDAP Server」軟體套件 或使用基於OpenLDAP的Docker容器。

  • 提供者類型：「Active Directory」（用於備援認證）
  • 使用者篩選器：(objectClass=inetOrgPerson)
  • 使用者DN格式：cn=username,ou=users,dc=example,dc=com

測試連線

請使用「測試連線」按鈕來驗證設定。 連線成功後，將顯示找到的使用者數量和一些範例使用者名稱。

疑難排解

問題解決方案
連線失敗請檢查主機、連接埠和SSL設定
繫結失敗請檢查繫結DN和密碼
找不到使用者請檢查基礎DN和使用者篩選器
地址未匯入OpenLDAP使用「street」而非「streetAddress」
