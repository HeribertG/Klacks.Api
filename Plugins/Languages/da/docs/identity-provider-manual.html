ID-udbyder manual

Introduktion

Ved at bruge ID-udbydere kan du autentificere brugere via eksterne katalogtjenester som LDAP, Active Directory, OAuth2/OpenID Connect.

Understøttede udbydertyper:

  • LDAP - Standard Lightweight Directory Access Protocol
  • Active Directory - Microsoft Active Directory
  • OAuth2 - OAuth 2.0 autorisationsramme
  • OpenID Connect - Identitetslag oven på OAuth 2.0

Klientimport

Hvis "Brug til klientimport" er aktiveret, importeres følgende data:

  • Efternavn - sn (efternavn) eller displayName
  • Fornavn - givenName
  • Titel - title (jobtitel)
  • Firma - company
  • Adresse - street/streetAddress, postalCode, l (by), st (region), c (land)
  • Køn - For Active Directory fra personalTitle (Herr/Frau)

Bemærk: For OpenLDAP sættes køn som standard til "Neutral" og kan rettes manuelt.

Tips

Brug af Synology NAS som LDAP-server

Du kan bruge en Synology NAS som LDAP-server. I dette tilfælde skal du installere pakken "LDAP Server" eller en Docker-container med OpenLDAP.

  • Udbydertype: "Active Directory" (til fallback-autentificering)
  • Brugerfilter: (objectClass=inetOrgPerson)
  • Bruger-DN format: cn=brugernavn,ou=users,dc=example,dc=com

Test af forbindelse

Brug knappen "Test forbindelse" for at bekræfte dine indstillinger. Ved vellykket forbindelse vises antallet af fundne brugere og nogle eksempelbrugernavne.
