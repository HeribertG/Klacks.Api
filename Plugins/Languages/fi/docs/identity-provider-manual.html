Identiteetintarjoajan käsikirja

Johdanto

Identiteetintarjoajat mahdollistavat käyttäjien todennuksen ulkoisten hakemistopalveluiden kuten LDAP:n, Active Directoryn tai OAuth2/OpenID Connectin kautta.

Tuetut tarjoajatyypit:

  • LDAP - Lightweight Directory Access Protocol
  • Active Directory - Microsoft Active Directory
  • OAuth2 - OAuth 2.0 -valtuutuskehys
  • OpenID Connect - Identiteettikerros OAuth 2.0:n päällä

Asiakastuonti

Kun "Käytä asiakastuontiin" on aktivoitu, seuraavat tiedot tuodaan:

  • Sukunimi - sn (sukunimi) tai displayName
  • Etunimi - givenName
  • Titteli - title (ammattinimike)
  • Yritys - company
  • Osoite - street/streetAddress, postalCode, l (kaupunki), st (lääni), c (maa)
  • Sukupuoli - personalTitle-kentästä (Herra/Rouva) Active Directoryssa

Huomautus: OpenLDAP:ssa sukupuoli on oletuksena "Neutraali" ja se voidaan korjata manuaalisesti.

Vinkkejä

Synology NAS LDAP-palvelimena

Synology NAS:ia voidaan käyttää LDAP-palvelimena. Tätä varten on asennettava "LDAP Server" -paketti tai Docker-kontti OpenLDAP:lla.

  • Tarjoajatyyppi: "Active Directory" (varatodennusta varten)
  • Käyttäjäsuodatin: (objectClass=inetOrgPerson)
  • Käyttäjä-DN-muoto: cn=käyttäjänimi,ou=users,dc=example,dc=com

Yhteyden testaaminen

Käytä "Testaa yhteys" -painiketta asetusten tarkistamiseen. Onnistuneen yhteyden yhteydessä näytetään löydettyjen käyttäjien määrä ja joitakin esimerkkikäyttäjänimiä.
