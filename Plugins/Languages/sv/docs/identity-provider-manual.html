Handbok för identitetsleverantör

Inledning

Identitetsleverantörer möjliggör autentisering av användare via externa katalogtjänster som LDAP, Active Directory eller OAuth2/OpenID Connect.

Stödda leverantörstyper:

  • LDAP - Lightweight Directory Access Protocol
  • Active Directory - Microsoft Active Directory
  • OAuth2 - OAuth 2.0-auktoriseringsramverk
  • OpenID Connect - Identitetslager ovanpå OAuth 2.0

Kundimport

När "Använd för kundimport" är aktiverat importeras följande data:

  • Efternamn - sn (efternamn) eller displayName
  • Förnamn - givenName
  • Titel - title (jobbtitel)
  • Företag - company
  • Adress - street/streetAddress, postalCode, l (stad), st (län), c (land)
  • Kön - Från personalTitle (Herr/Fru) i Active Directory

Obs: I OpenLDAP är könet som standard "Neutralt" och kan korrigeras manuellt.

Tips

Synology NAS som LDAP-server

En Synology NAS kan användas som LDAP-server. För detta måste paketet "LDAP Server" eller en Docker-container med OpenLDAP installeras.

  • Leverantörstyp: "Active Directory" (för reservautentisering)
  • Användarfilter: (objectClass=inetOrgPerson)
  • Användar-DN-format: cn=användarnamn,ou=users,dc=example,dc=com

Testa anslutningen

Använd knappen "Testa anslutning" för att verifiera konfigurationen. Vid en lyckad anslutning visas antalet hittade användare och några exempelanvändarnamn.
