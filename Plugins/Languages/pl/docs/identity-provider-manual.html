Podręcznik dostawcy tożsamości

Wprowadzenie

Dostawcy tożsamości umożliwiają uwierzytelnianie użytkowników za pośrednictwem zewnętrznych usług katalogowych, takich jak LDAP, Active Directory lub OAuth2/OpenID Connect.

Obsługiwane typy dostawców:

  • LDAP - Lightweight Directory Access Protocol
  • Active Directory - Microsoft Active Directory
  • OAuth2 - Framework autoryzacji OAuth 2.0
  • OpenID Connect - Warstwa tożsamości na bazie OAuth 2.0

Import klientów

Gdy opcja "Użyj do importu klientów" jest aktywowana, importowane są następujące dane:

  • Nazwisko - sn (nazwisko) lub displayName
  • Imię - givenName
  • Tytuł - title (tytuł zawodowy)
  • Firma - company
  • Adres - street/streetAddress, postalCode, l (miasto), st (województwo), c (kraj)
  • Płeć - Na podstawie personalTitle (Pan/Pani) w Active Directory

Uwaga: W OpenLDAP płeć jest domyślnie ustawiona na "Neutralna" i może być ręcznie skorygowana.

Wskazówki

Synology NAS jako serwer LDAP

Synology NAS może być używany jako serwer LDAP. W tym celu należy zainstalować pakiet "LDAP Server" lub kontener Docker z OpenLDAP.

  • Typ dostawcy: "Active Directory" (dla uwierzytelniania zapasowego)
  • Filtr użytkowników: (objectClass=inetOrgPerson)
  • Format DN użytkownika: cn=nazwa_użytkownika,ou=users,dc=example,dc=com

Testowanie połączenia

Użyj przycisku "Testuj połączenie", aby zweryfikować konfigurację. W przypadku pomyślnego połączenia wyświetlana jest liczba znalezionych użytkowników oraz kilka przykładowych nazw użytkowników.
