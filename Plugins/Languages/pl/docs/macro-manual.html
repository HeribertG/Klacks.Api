Podręcznik skryptów makr

Wprowadzenie

Edytor makr umożliwia pisanie skryptów, które służą jako podstawa obliczeniowa dla usług, czynności roboczych itp.

Przykłady zastosowania:

  • Dokładne obliczanie godzin urlopu lub służby wojskowej na podstawie wymiaru zatrudnienia
  • Obliczanie dodatków za pracę nocną, weekendy i dni świąteczne
  • Elastyczne dostosowanie do nowych wymagań

Uwaga: Interpreter skryptów działa w bezpiecznym środowisku piaskownicy bez dostępu do systemu plików ani sieci. Wykonanie złośliwego kodu nie jest możliwe.

Wszystkie zmienne i stałe są beztypowe.

Struktury sterujące

Ważne: Słowa kluczowe takie jak ENDIF, ENDFUNCTION, ENDSUB muszą być pisane łącznie (nie END IF).

IF-THEN-ELSE

IF x > 10 THEN
    OUTPUT 1, "duży"
ELSEIF x > 5 THEN
    OUTPUT 1, "średni"
ELSE
    OUTPUT 1, "mały"
ENDIF

IF w jednej linii

IF x > 10 THEN OUTPUT 1, "duży" ENDIF

SELECT CASE

SELECT CASE weekday
    CASE 6, 7
        OUTPUT 1, "Weekend"
    CASE 1, 2, 3, 4, 5
        OUTPUT 1, "Dzień roboczy"
    CASE ELSE
        OUTPUT 1, "Nieznany"
END SELECT

Pętla FOR-NEXT

FOR i = 1 TO 10
    suma += i
NEXT

FOR i = 10 TO 1 STEP -1
    IF x > y THEN EXIT FOR
NEXT

Pętla DO-LOOP

DO WHILE a > 0
    a -= 1
LOOP

DO
    x += 1
LOOP UNTIL x >= 10

Funkcje i procedury

Ważne: Funkcje (FUNCTION) i procedury (SUB) muszą być zdefiniowane przed pierwszym wywołaniem. Interpreter czyta kod od góry do dołu: funkcja lub procedura jest znana dopiero po przetworzeniu jej definicji.

SUB (procedura bez wartości zwracanej)

SUB oblicz(a, b)
    IF a = b THEN EXIT SUB ENDIF
    OUTPUT 1, a + b
ENDSUB

FUNCTION (funkcja z wartością zwracaną)

FUNCTION podwoj(x)
    podwoj = x * 2
ENDFUNCTION

DIM wynik
wynik = podwoj(21)
OUTPUT 1, wynik

Przykład: Obliczanie dodatków

Oblicza dodatki za pracę nocną, dni świąteczne i weekendy z prawidłową obsługą zmian przekraczających północ (podział na segmenty).

IMPORT hour, fromhour, untilhour
IMPORT weekday, holiday, holidaynextday
IMPORT nightrate, holidayrate, sarate, sorate

FUNCTION CalcSegment(StartHour, EndHour, IsHoliday, WeekdayNum)
    DIM SegmentHours, NightHours, NonNightHours
    DIM NRate, DRate, HasHoliday, IsSaturday, IsSunday

    SegmentHours = EndHour - StartHour
    IF SegmentHours < 0 THEN SegmentHours = SegmentHours + 24 ENDIF

    ' Oblicz godziny nocne (23:00-06:00)
    NightHours = 0
    IF StartHour < 6 THEN NightHours = NightHours + IIF(EndHour < 6, EndHour, 6) - StartHour ENDIF
    IF EndHour > 23 OrElse EndHour < StartHour THEN NightHours = NightHours + IIF(StartHour > 23, 24 - StartHour, 24 - 23) ENDIF
    NonNightHours = SegmentHours - NightHours

    HasHoliday = IsHoliday
    IsSaturday = WeekdayNum = 6    ' So = 6 (ISO-8601)
    IsSunday = WeekdayNum = 7      ' Nd = 7 (ISO-8601)

    ' Najwyższa stawka dla godzin nocnych
    NRate = nightrate
    IF HasHoliday AndAlso holidayrate > NRate THEN NRate = holidayrate ENDIF
    IF IsSaturday AndAlso sarate > NRate THEN NRate = sarate ENDIF
    IF IsSunday AndAlso sorate > NRate THEN NRate = sorate ENDIF

    ' Najwyższa stawka dla godzin dziennych
    DRate = 0
    IF HasHoliday AndAlso holidayrate > DRate THEN DRate = holidayrate ENDIF
    IF IsSaturday AndAlso sarate > DRate THEN DRate = sarate ENDIF
    IF IsSunday AndAlso sorate > DRate THEN DRate = sorate ENDIF

    CalcSegment = NightHours * NRate + NonNightHours * DRate
ENDFUNCTION

DIM TotalBonus, WeekdayNextDay

' Oblicz dzień tygodnia następnego dnia (ISO-8601)
WeekdayNextDay = (weekday MOD 7) + 1

IF untilhour <= fromhour THEN
    ' Zmiana przekraczająca północ - 2 segmenty
    TotalBonus = CalcSegment(fromhour, 24, holiday, weekday)
    TotalBonus = TotalBonus + CalcSegment(0, untilhour, holidaynextday, WeekdayNextDay)
ELSE
    ' Normalna zmiana dzienna
    TotalBonus = CalcSegment(fromhour, untilhour, holiday, weekday)
ENDIF

OUTPUT 1, Round(TotalBonus, 2)

Wyjaśnienie

  • Podział na segmenty: Dla zmian przekraczających północ (np. 22:00-06:00) zmiana jest dzielona na dwa segmenty
  • CalcSegment: Oblicza dodatek dla segmentu z uwzględnieniem dodatków nocnych, świątecznych i weekendowych
  • Osobne stawki dla soboty/niedzieli: sarate i sorate pozwalają na różne dodatki dla soboty i niedzieli
  • Najwyższy dodatek: Zawsze stosowany jest najwyższy obowiązujący dodatek
  • holidaynextday: Uwzględnia, czy następny dzień jest dniem świątecznym (ważne dla zmian nocnych)
