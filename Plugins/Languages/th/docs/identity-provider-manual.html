คู่มือผู้ให้บริการยืนยันตัวตน

บทนำ

ผู้ให้บริการยืนยันตัวตนช่วยให้สามารถยืนยันตัวตนผู้ใช้ผ่านบริการไดเรกทอรีภายนอก เช่น LDAP, Active Directory, OAuth2/OpenID Connect

ประเภทผู้ให้บริการที่รองรับ:

  • LDAP - โปรโตคอลเข้าถึงไดเรกทอรีน้ำหนักเบามาตรฐาน
  • Active Directory - Microsoft Active Directory
  • OAuth2 - กรอบการทำงานการอนุญาต OAuth 2.0
  • OpenID Connect - ชั้นยืนยันตัวตนบน OAuth 2.0

การนำเข้าลูกค้า

หากเปิดใช้งาน "ใช้สำหรับนำเข้าลูกค้า" ข้อมูลต่อไปนี้จะถูกนำเข้า:

  • นามสกุล - sn (นามสกุล) หรือ displayName
  • ชื่อ - givenName
  • ตำแหน่ง - title (ตำแหน่งงาน)
  • บริษัท - company
  • ที่อยู่ - street/streetAddress, postalCode, l (เมือง), st (จังหวัด), c (ประเทศ)
  • เพศ - สำหรับ Active Directory จะดึงจาก personalTitle (Herr/Frau)

หมายเหตุ: สำหรับ OpenLDAP เพศจะเป็น "กลาง" โดยค่าเริ่มต้น และสามารถแก้ไขด้วยตนเองได้

เคล็ดลับ

ใช้ Synology NAS เป็นเซิร์ฟเวอร์ LDAP

คุณสามารถใช้ Synology NAS เป็นเซิร์ฟเวอร์ LDAP ได้ ในกรณีนี้ คุณต้องติดตั้ง แพ็กเกจ "LDAP Server" หรือคอนเทนเนอร์ Docker ที่ใช้ OpenLDAP

  • ประเภทผู้ให้บริการ: "Active Directory" (สำหรับการยืนยันตัวตนสำรอง)
  • ตัวกรองผู้ใช้: (objectClass=inetOrgPerson)
  • รูปแบบ User DN: cn=username,ou=users,dc=example,dc=com

ทดสอบการเชื่อมต่อ

ใช้ปุ่ม "ทดสอบการเชื่อมต่อ" เพื่อตรวจสอบการตั้งค่า เมื่อเชื่อมต่อสำเร็จ จะแสดงจำนวนผู้ใช้ที่พบและชื่อผู้ใช้ตัวอย่างบางส่วน

การแก้ไขปัญหา
