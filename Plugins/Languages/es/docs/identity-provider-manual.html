Manual del Proveedor de Identidad

Introducción

Los proveedores de identidad permiten la autenticación de usuarios a través de servicios de directorio externos como LDAP, Active Directory u OAuth2/OpenID Connect.

Tipos de proveedores compatibles:

  • LDAP - Protocolo estándar de acceso ligero a directorios
  • Active Directory - Microsoft Active Directory
  • OAuth2 - Marco de autorización OAuth 2.0
  • OpenID Connect - Capa de identidad sobre OAuth 2.0

Importación de clientes

Cuando está activado "Usar para importación de clientes", se importan los siguientes datos:

  • Apellido - sn (apellido) o displayName
  • Nombre - givenName
  • Título - title (título del puesto)
  • Empresa - company
  • Dirección - street/streetAddress, postalCode, l (ciudad), st (provincia), c (país)
  • Género - A partir de personalTitle (Sr./Sra.) en Active Directory

Nota: En OpenLDAP, el género es por defecto "Neutro" y puede corregirse manualmente.

Consejos

NAS Synology como servidor LDAP

Un NAS Synology puede utilizarse como servidor LDAP. Para ello, es necesario instalar el paquete "LDAP Server" o un contenedor Docker con OpenLDAP.

  • Tipo de proveedor: "Active Directory" (para autenticación de reserva)
  • Filtro de usuario: (objectClass=inetOrgPerson)
  • Formato DN de usuario: cn=nombre_usuario,ou=users,dc=example,dc=com

Probar la conexión

Utilice el botón "Probar conexión" para verificar la configuración. Si la conexión es exitosa, se muestran el número de usuarios encontrados y algunos nombres de usuario de ejemplo.
