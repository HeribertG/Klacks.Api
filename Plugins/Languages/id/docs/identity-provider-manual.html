Panduan Identity Provider

Pendahuluan

Dengan menggunakan Identity Provider, Anda dapat melakukan autentikasi pengguna melalui layanan direktori eksternal seperti LDAP, Active Directory, OAuth2/OpenID Connect.

Jenis penyedia yang didukung:

  • LDAP - Protokol Akses Direktori Ringan Standar
  • Active Directory - Microsoft Active Directory
  • OAuth2 - Kerangka Otorisasi OAuth 2.0
  • OpenID Connect - Lapisan identitas di atas OAuth 2.0

Impor Klien

Jika "Gunakan untuk impor klien" diaktifkan, data berikut akan diimpor:

  • Nama belakang - sn (nama keluarga) atau displayName
  • Nama depan - givenName
  • Jabatan - title (nama jabatan)
  • Perusahaan - company
  • Alamat - street/streetAddress, postalCode, l (kota), st (provinsi), c (negara)
  • Jenis kelamin - Untuk Active Directory, diambil dari personalTitle (Herr/Frau)

Catatan: Untuk OpenLDAP, jenis kelamin secara default ditetapkan "Netral" dan dapat diperbaiki secara manual.

Tips

Menggunakan Synology NAS sebagai Server LDAP

Anda dapat menggunakan Synology NAS sebagai server LDAP. Dalam hal ini, Anda perlu menginstal paket "LDAP Server" atau container Docker dengan OpenLDAP.

  • Jenis penyedia: "Active Directory" (untuk autentikasi fallback)
  • Filter pengguna: (objectClass=inetOrgPerson)
  • Format DN pengguna: cn=username,ou=users,dc=example,dc=com

Menguji Koneksi

Gunakan tombol "Uji koneksi" untuk memverifikasi pengaturan. Jika koneksi berhasil, jumlah pengguna yang ditemukan dan beberapa contoh nama pengguna akan ditampilkan.
