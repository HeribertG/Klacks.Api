Manual do fornecedor de identidade

Introdução

Os fornecedores de identidade permitem autenticar utilizadores através de serviços de diretório externos como LDAP, Active Directory, OAuth2/OpenID Connect.

Tipos de fornecedores suportados:

  • LDAP - Protocolo leve de acesso a diretórios
  • Active Directory - Microsoft Active Directory
  • OAuth2 - Framework de autorização OAuth 2.0
  • OpenID Connect - Camada de identidade sobre OAuth 2.0

Importação de clientes

Se "Usar para importação de clientes" estiver ativado, os seguintes dados são importados:

  • Apelido - sn (apelido) ou displayName
  • Nome - givenName
  • Cargo - title (título profissional)
  • Empresa - company
  • Morada - street/streetAddress, postalCode, l (localidade), st (distrito), c (país)
  • Género - No Active Directory, obtido de personalTitle (Herr/Frau)

Nota: No OpenLDAP, o género é predefinido como "Neutro" e pode ser corrigido manualmente.

Dicas

Usar Synology NAS como servidor LDAP

Pode usar um Synology NAS como servidor LDAP. Nesse caso, é necessário instalar o pacote "LDAP Server" ou um contentor Docker com OpenLDAP.

  • Tipo de fornecedor: "Active Directory" (para autenticação de fallback)
  • Filtro de utilizadores: (objectClass=inetOrgPerson)
  • Formato do DN do utilizador: cn=username,ou=users,dc=example,dc=com

Testar a ligação

Use o botão "Testar ligação" para verificar as configurações. Se a ligação for bem-sucedida, será apresentado o número de utilizadores encontrados e alguns nomes de utilizadores de exemplo.
