מדריך ספק זהות

מבוא

באמצעות ספקי זהות ניתן לבצע אימות משתמשים דרך שירותי ספרייה חיצוניים כגון LDAP, Active Directory, OAuth2/OpenID Connect.

סוגי ספקים נתמכים:

  • LDAP - פרוטוקול גישה קלת משקל לספרייה
  • Active Directory - Microsoft Active Directory
  • OAuth2 - מסגרת הרשאות OAuth 2.0
  • OpenID Connect - שכבת זהות מעל OAuth 2.0

ייבוא לקוחות

כאשר "השתמש לייבוא לקוחות" מופעל, הנתונים הבאים מיובאים:

  • שם משפחה - sn (שם משפחה) או displayName
  • שם פרטי - givenName
  • תפקיד - title (שם התפקיד)
  • חברה - company
  • כתובת - street/streetAddress, postalCode, l (עיר), st (מדינה/מחוז), c (מדינה)
  • מין - ב-Active Directory, נלקח מ-personalTitle (Herr/Frau)

הערה: ב-OpenLDAP, המין מוגדר כברירת מחדל כ"ניטרלי" וניתן לתקן ידנית.

טיפים

שימוש ב-Synology NAS כשרת LDAP

ניתן להשתמש ב-Synology NAS כשרת LDAP. במקרה זה, יש להתקין את חבילת "שרת LDAP" או מכולת Docker עם OpenLDAP.

  • סוג ספק: "Active Directory" (לאימות חלופי)
  • מסנן משתמשים: (objectClass=inetOrgPerson)
  • פורמט DN משתמש: cn=username,ou=users,dc=example,dc=com

בדיקת חיבור

השתמש בכפתור "בדוק חיבור" כדי לאמת את ההגדרות. אם החיבור מצליח, יוצגו מספר המשתמשים שנמצאו ומספר שמות משתמשים לדוגמה.
