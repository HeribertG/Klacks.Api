Identitetsleverandør - Manual

Innledning

Ved bruk av identitetsleverandører kan brukerautentisering utføres via eksterne katalogtjenester som LDAP, Active Directory, OAuth2/OpenID Connect.

Støttede leverandørtyper:

  • LDAP - Standard Lightweight Directory Access Protocol
  • Active Directory - Microsoft Active Directory
  • OAuth2 - OAuth 2.0-autorisasjonsrammeverk
  • OpenID Connect - Identitetslag over OAuth 2.0

Klientimport

Hvis "Bruk for klientimport" er aktivert, importeres følgende data:

  • Etternavn - sn (etternavn) eller displayName
  • Fornavn - givenName
  • Stilling - title (stillingstittel)
  • Firma - company
  • Adresse - street/streetAddress, postalCode, l (by), st (fylke), c (land)
  • Kjønn - For Active Directory, hentes fra personalTitle (Herr/Frau)

Merk: For OpenLDAP settes kjønn som standard til "Nøytral" og kan korrigeres manuelt.

Tips

Bruk av Synology NAS som LDAP-server

Du kan bruke en Synology NAS som LDAP-server. I så fall må du installere "LDAP Server"-pakken eller en Docker-container med OpenLDAP.

  • Leverandørtype: "Active Directory" (for reserveautentisering)
  • Brukerfilter: (objectClass=inetOrgPerson)
  • Bruker-DN-format: cn=brukernavn,ou=users,dc=example,dc=com

Test av tilkobling

Bruk knappen "Test tilkobling" for å bekrefte innstillingene. Ved vellykket tilkobling vises antall brukere som ble funnet og noen eksempelbrukernavn.
