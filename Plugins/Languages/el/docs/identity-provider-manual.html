Εγχειρίδιο παρόχου ταυτότητας

Εισαγωγή

Οι πάροχοι ταυτότητας επιτρέπουν την αυθεντικοποίηση χρηστών μέσω εξωτερικών υπηρεσιών καταλόγου όπως LDAP, Active Directory ή OAuth2/OpenID Connect.

Υποστηριζόμενοι τύποι παρόχων:

  • LDAP - Lightweight Directory Access Protocol
  • Active Directory - Microsoft Active Directory
  • OAuth2 - Πλαίσιο εξουσιοδότησης OAuth 2.0
  • OpenID Connect - Επίπεδο ταυτότητας πάνω από OAuth 2.0

Εισαγωγή πελατών

Όταν είναι ενεργοποιημένη η επιλογή "Χρήση για εισαγωγή πελατών", εισάγονται τα ακόλουθα δεδομένα:

  • Επώνυμο - sn (επώνυμο) ή displayName
  • Όνομα - givenName
  • Τίτλος - title (τίτλος θέσης)
  • Εταιρεία - company
  • Διεύθυνση - street/streetAddress, postalCode, l (πόλη), st (περιφέρεια), c (χώρα)
  • Φύλο - Από το personalTitle (Κύριος/Κυρία) στο Active Directory

Σημείωση: Στο OpenLDAP το φύλο είναι προεπιλεγμένα "Ουδέτερο" και μπορεί να διορθωθεί χειροκίνητα.

Συμβουλές

Synology NAS ως διακομιστής LDAP

Ένα Synology NAS μπορεί να χρησιμοποιηθεί ως διακομιστής LDAP. Για αυτό πρέπει να εγκατασταθεί το πακέτο "LDAP Server" ή ένα Docker container με OpenLDAP.

  • Τύπος παρόχου: "Active Directory" (για εφεδρική αυθεντικοποίηση)
  • Φίλτρο χρηστών: (objectClass=inetOrgPerson)
  • Μορφή DN χρήστη: cn=όνομα_χρήστη,ou=users,dc=example,dc=com

Δοκιμή σύνδεσης

Χρησιμοποιήστε το κουμπί "Δοκιμή σύνδεσης" για να επαληθεύσετε τη ρύθμιση. Σε περίπτωση επιτυχούς σύνδεσης, εμφανίζεται ο αριθμός των χρηστών που βρέθηκαν και μερικά παραδείγματα ονομάτων χρηστών.
