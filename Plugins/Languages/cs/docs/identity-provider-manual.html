Příručka poskytovatele identity

Úvod

Poskytovatelé identity umožňují ověřování uživatelů prostřednictvím externích adresářových služeb, jako jsou LDAP, Active Directory nebo OAuth2/OpenID Connect.

Podporované typy poskytovatelů:

  • LDAP - Lightweight Directory Access Protocol
  • Active Directory - Microsoft Active Directory
  • OAuth2 - Autorizační framework OAuth 2.0
  • OpenID Connect - Vrstva identity nad OAuth 2.0

LDAP:             (objectClass=person)

Active Directory: (&(objectClass=user)(objectCategory=person))
                  (&(objectClass=user)(!(userAccountControl:1.2.840.113556.1.4.803:=2)))

Import klientů

Když je aktivováno "Použít pro import klientů", importují se následující údaje:

  • Příjmení - sn (příjmení) nebo displayName
  • Jméno - givenName
  • Titul - title (pracovní pozice)
  • Firma - company
  • Adresa - street/streetAddress, postalCode, l (město), st (kraj), c (země)
  • Pohlaví - Z personalTitle (Pan/Paní) v Active Directory

Poznámka: V OpenLDAP je pohlaví standardně nastaveno na "Neutrální" a může být ručně opraveno.

Tipy

Synology NAS jako LDAP server

Synology NAS lze použít jako LDAP server. K tomu je nutné nainstalovat balíček "LDAP Server" nebo Docker kontejner s OpenLDAP.

  • Typ poskytovatele: "Active Directory" (pro záložní ověření)
  • Filtr uživatelů: (objectClass=inetOrgPerson)
  • Formát DN uživatele: cn=uživatelské_jméno,ou=users,dc=example,dc=com

Testování připojení

Použijte tlačítko "Testovat připojení" pro ověření konfigurace. Při úspěšném připojení se zobrazí počet nalezených uživatelů a několik ukázkových uživatelských jmen.
