LINE Messaging API 설정

사전 준비

LINE을 통해 메시지를 보내고 받으려면 Messaging API가 활성화된 LINE 공식 계정이 필요합니다. LINE은 일본, 대만, 태국에서 대표적인 메신저입니다.

참고: 매월 처음 500건의 메시지는 무료입니다. 그 이후에는 요금이 부과됩니다. 자세한 정보는 developers.line.biz를 방문하세요.

1단계 — LINE 공식 계정 및 Messaging API 채널 생성

  1. developers.line.biz에서 가입합니다.
  2. LINE 공식 계정을 생성하고 Messaging API를 활성화합니다.
  3. LINE Developers Console에서 Messaging API 채널을 생성합니다.
  4. Channel Access Token (장기)을 확인합니다.
  5. 채널 설정에서 Channel Secret을 확인합니다.

2단계 — Klacks에서 LINE 구성

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. 추가를 클릭합니다.
  3. 다음과 같이 필드를 입력합니다:
필드설명예
이름고유한 내부 이름line-messaging
표시 이름사용자 인터페이스에 표시됨LINE
프로바이더 유형LINE 선택LINE
Channel Access TokenLINE Developers Console의 장기 토큰eyJhbGci...
Channel Secret채널 설정의 Channel Secreta1b2c3d4...
활성화프로바이더 활성화/비활성화예
  1. 저장을 클릭합니다.
  2. 프로바이더 목록의 테스트 버튼으로 연결을 테스트합니다.

3단계 — 챗봇(Klacksy)을 통한 메시지 전송

LINE 프로바이더가 구성되고 활성화되면 챗봇 Klacksy가 메시지를 보낼 수 있습니다. LINE User ID를 알 필요 없이 직원 또는 고객의 이름만 사용하면 됩니다: 

"홍길동에게 메시지 보내기: 내일 오후 2시에 예약이 있습니다."
"김영희에게 메시지 보내기: 근무가 교대되었습니다."
"이철수에게 월요일 근무가 취소되었다고 알려주세요."

Klacksy가 자동으로 수신자의 LINE User ID를 조회하고 구성된 LINE 프로바이더를 사용합니다. 이 과정은 사용자에게 완전히 투명합니다 — 이름으로 작업하면 Klacksy가 나머지를 처리합니다.

전제 조건: 수신자의 LINE User ID가 Klacks에 저장되어 있어야 합니다 (주소 → 연락처 정보).

4단계 — 수신 메시지 (Webhook)

수신 LINE 메시지를 Klacks에 표시하려면 LINE Developers Console에서 Webhook을 구성해야 합니다.

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. LINE 프로바이더의 자동 생성된 Webhook 시크릿을 확인합니다.
  3. LINE Developers Console에서 Webhook URL을 구성합니다: 
    https://{your-domain}/api/messaging/webhook/line
  4. 수신 메시지가 이제 메시징 수신함에 실시간으로 표시됩니다.

문제 해결

문제가능한 원인해결 방법
연결 테스트 실패 Channel Access Token 또는 Channel Secret이 잘못됨 LINE Developers Console에서 자격 증명을 확인하세요
메시지가 전달되지 않음 LINE User ID가 유효하지 않음 LINE User ID가 올바르게 저장되어 있는지 확인하세요
수신자가 메시지를 받지 못함 수신자가 공식 계정을 추가하지 않음 수신자가 LINE 공식 계정을 친구로 추가해야 합니다
수신 메시지 누락 Webhook이 구성되지 않음 LINE Developers Console에서 Webhook URL을 확인하세요
메시지 제한 도달 월간 무료 할당량 (500건의 메시지) 소진 LINE 플랜을 업그레이드하거나 다음 달을 기다리세요
