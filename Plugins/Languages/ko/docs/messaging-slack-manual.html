Slack 설정

사전 준비

Klacks를 통해 Slack으로 메시지를 보내고 받으려면 Slack 워크스페이스와 적절한 권한을 가진 Slack 앱이 필요합니다.

참고: 무료 Slack 플랜은 최대 10,000개의 메시지 기록을 허용합니다. 전체 API 사용을 위해서는 Pro 플랜을 권장합니다.

1단계 — Slack 앱 생성

  1. api.slack.com/apps를 열고 Create New App을 클릭합니다.
  2. From scratch를 선택하고 이름을 입력합니다 (예: Klacks Bot).
  3. 앱을 설치할 워크스페이스를 선택합니다.
  4. OAuth & Permissions로 이동하고 다음 Bot Token Scopes를 추가합니다:
    • chat:write — 메시지 전송용
    • users:read — 사용자 정보 읽기용
  5. Install to Workspace를 클릭하고 권한을 확인합니다.
  6. Bot User OAuth Token을 확인합니다 (xoxb-로 시작).
  7. Basic Information으로 이동하고 Signing Secret을 확인합니다.

2단계 — Klacks에서 Slack 구성

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. 추가를 클릭합니다.
  3. 다음과 같이 필드를 입력합니다:
필드설명예
이름고유한 내부 이름slack-main
표시 이름사용자 인터페이스에 표시됨Slack
프로바이더 유형Slack 선택Slack
봇 토큰Bot User OAuth Token (xoxb-로 시작)xoxb-1234567890-abcdef...
Signing SecretBasic Information의 Signing Secreta1b2c3d4e5f6...
기본 채널특정 수신자가 없는 메시지의 기본 채널#general
활성화프로바이더 활성화/비활성화예
  1. 저장을 클릭합니다.
  2. 프로바이더 목록의 테스트 버튼으로 연결을 테스트합니다.

3단계 — 챗봇(Klacksy)을 통한 메시지 전송

Slack 프로바이더가 구성되고 활성화되면 챗봇 Klacksy가 Slack을 통해 메시지를 보낼 수 있습니다. 직원의 이름 또는 채널 이름을 사용하세요: 

"홍길동에게 메시지 보내기: 내일 오후 2시에 예약이 있습니다."
"김영희에게 메시지 보내기: 근무가 교대되었습니다."
"#팀-공지에 게시하기: 다음 주 일정이 준비되었습니다."

Klacksy가 자동으로 연결된 Slack 사용자를 조회하고 메시지를 보냅니다. 채널 메시지의 경우 지정된 채널에 직접 게시합니다. 이 과정은 사용자에게 완전히 투명합니다 — 이름으로 작업하면 Klacksy가 나머지를 처리합니다.

전제 조건: Slack 사용자가 해당 Klacks 연락처에 연결되어 있어야 합니다 (주소 → 연락처 정보). 채널 메시지의 경우 앱이 해당 채널에 추가되어 있어야 합니다.

4단계 — 수신 메시지 (Webhook)

수신 Slack 메시지를 Klacks에 표시하려면 Event Subscriptions를 구성해야 합니다.

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. Slack 프로바이더의 자동 생성된 Webhook 시크릿을 확인합니다.
  3. Slack 앱의 Event Subscriptions에서 요청 URL을 구성합니다: 
    https://{your-domain}/api/messaging/webhook/slack
  4. 수신 메시지가 이제 메시징 수신함에 실시간으로 표시됩니다.

문제 해결

문제가능한 원인해결 방법
연결 테스트 실패 봇 토큰 또는 Signing Secret이 잘못됨 Slack 앱 구성에서 자격 증명을 확인하세요
메시지가 전달되지 않음 앱이 워크스페이스에 설치되지 않음 OAuth & Permissions를 통해 앱을 다시 설치하세요
수신자가 메시지를 받지 못함 Slack 사용자가 Klacks 연락처에 연결되지 않음 주소 → 연락처 정보에서 Slack 사용자를 연결하세요
채널 메시지 실패 봇이 채널에 추가되지 않음 Slack 채널에 앱을 추가하세요 (채널 → 통합)
수신 메시지 누락 Event Subscriptions가 구성되지 않음 Slack 앱의 Event Subscriptions에서 요청 URL을 확인하세요
