ID 프로바이더 매뉴얼

소개

ID 프로바이더를 사용하면 LDAP, Active Directory, OAuth2/OpenID Connect 등 외부 디렉토리 서비스를 통해 사용자 인증을 수행할 수 있습니다.

지원 프로바이더 유형:

  • LDAP - 표준 경량 디렉토리 액세스 프로토콜
  • Active Directory - Microsoft Active Directory
  • OAuth2 - OAuth 2.0 인가 프레임워크
  • OpenID Connect - OAuth 2.0 기반 ID 레이어

클라이언트 가져오기

"클라이언트 가져오기에 사용"을 활성화한 경우 다음 데이터를 가져옵니다:

  • 성 - sn (성) 또는 displayName
  • 이름 - givenName
  • 직책 - title (직책명)
  • 회사 - company
  • 주소 - street/streetAddress, postalCode, l (시/구), st (도/광역시), c (국가)
  • 성별 - Active Directory의 경우 personalTitle (Herr/Frau)에서 가져옴

참고: OpenLDAP의 경우 성별은 기본적으로 "중립"으로 설정되며 수동으로 수정할 수 있습니다.

팁

Synology NAS를 LDAP 서버로 사용

Synology NAS를 LDAP 서버로 사용할 수 있습니다. 이 경우 "LDAP 서버" 패키지 또는 OpenLDAP를 사용한 Docker 컨테이너를 설치해야 합니다.

  • 프로바이더 유형: "Active Directory" (폴백 인증용)
  • 사용자 필터: (objectClass=inetOrgPerson)
  • 사용자 DN 형식: cn=username,ou=users,dc=example,dc=com

연결 테스트

"연결 테스트" 버튼을 사용하여 설정을 확인하세요. 연결이 성공하면 발견된 사용자 수와 몇 가지 샘플 사용자명이 표시됩니다.
