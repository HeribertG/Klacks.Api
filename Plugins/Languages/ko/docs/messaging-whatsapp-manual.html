WhatsApp Business 설정

사전 준비

Klacks를 통해 WhatsApp 메시지를 보내고 받으려면 Meta의 WhatsApp Business API에 대한 액세스가 필요합니다. Meta 개발자 포털을 통해 관리됩니다.

참고: WhatsApp Business API는 인증된 비즈니스 신원이 필요합니다. 매월 처음 1,000건의 대화는 무료입니다.

1단계 — WhatsApp Business API 설정

  1. developers.facebook.com (Meta 개발자 포털)에서 계정을 생성합니다.
  2. 새 앱을 만들고 Business 유형을 선택합니다.
  3. 앱에 WhatsApp 제품을 추가합니다.
  4. WhatsApp 메시징용 전화번호를 등록합니다.
  5. Meta Business Suite에서 비즈니스 신원을 인증합니다.
  6. 개발자 포털에서 다음 값을 확인합니다:
    • 액세스 토큰 — WhatsApp → API 설정에서
    • 전화번호 ID — 등록된 번호의 ID
    • 비즈니스 계정 ID — Meta Business Suite에서

2단계 — Klacks에서 WhatsApp 구성

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. 추가를 클릭합니다.
  3. 프로바이더 유형으로 WhatsApp을 선택합니다.
  4. 다음과 같이 필드를 입력합니다:
필드설명예
이름고유한 내부 이름whatsapp-business
표시 이름사용자 인터페이스에 표시됨WhatsApp
프로바이더 유형WhatsApp 선택WhatsApp
액세스 토큰Meta 개발자 포털의 영구 액세스 토큰EAABs...
전화번호 ID등록된 WhatsApp 전화번호의 ID106540352...
비즈니스 계정 IDMeta Business Suite의 WhatsApp Business 계정 ID102939...
활성화프로바이더 활성화/비활성화예
  1. 저장을 클릭합니다.
  2. 프로바이더 목록의 테스트 버튼으로 연결을 테스트합니다.

3단계 — 챗봇(Klacksy)을 통한 메시지 전송

WhatsApp 프로바이더가 구성되고 활성화되면 챗봇 Klacksy가 메시지를 보낼 수 있습니다. 전화번호를 알 필요 없이 직원 또는 고객의 이름만 사용하면 됩니다: 

"홍길동에게 WhatsApp 보내기: 내일 오후 2시에 예약이 있습니다."
"김영희에게 WhatsApp으로 메시지 보내기: 근무가 교대되었습니다."
"이철수에게 WhatsApp으로 월요일 근무가 취소되었다고 알려주세요."

Klacksy가 자동으로 수신자의 저장된 휴대전화 번호를 조회하고 구성된 WhatsApp 프로바이더를 사용합니다. 이 과정은 사용자에게 완전히 투명합니다 — 이름으로 작업하면 Klacksy가 나머지를 처리합니다.

전제 조건: 수신자의 휴대전화 번호가 Klacks에 저장되어 있어야 합니다 (주소 → 연락처 정보). 번호가 WhatsApp에 등록되어 있어야 합니다.

4단계 — 수신 메시지 (Webhook)

수신 WhatsApp 메시지를 Klacks에 표시하려면 Meta 개발자 포털에서 Webhook을 구성해야 합니다.

  1. Meta 개발자 포털에서 앱 → WhatsApp → 구성으로 이동합니다.
  2. 다음을 Webhook URL로 입력합니다: 
    https://{your-domain}/api/messaging/webhook/whatsapp
  3. messages 필드를 구독합니다.
  4. 수신 WhatsApp 메시지가 이제 메시징 수신함에 실시간으로 표시됩니다.

문제 해결

문제가능한 원인해결 방법
연결 테스트 실패 액세스 토큰이 유효하지 않거나 만료됨 Meta 개발자 포털에서 새 영구 토큰을 생성하세요
메시지가 전달되지 않음 전화번호 ID 또는 비즈니스 계정 ID가 잘못됨 Meta 개발자 포털의 WhatsApp → API 설정에서 ID를 확인하세요
수신자가 메시지를 받지 못함 수신자의 해당 번호에 WhatsApp이 없음 휴대전화 번호가 WhatsApp에 등록되어 있는지 확인하세요
오류 403 — 권한 거부 비즈니스 신원이 인증되지 않음 Meta Business Suite에서 비즈니스를 인증하세요
수신 메시지 누락 Webhook이 구성되지 않았거나 URL에 접근할 수 없음 Meta 개발자 포털에서 Webhook URL과 SSL 인증서를 확인하세요
메시지 제한 도달 월간 할당량 (1,000건의 대화) 소진 다음 달을 기다리거나 Meta에 더 높은 한도를 요청하세요
