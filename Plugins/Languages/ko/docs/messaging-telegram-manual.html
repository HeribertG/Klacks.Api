Telegram 봇 설정

사전 준비

Klacks를 통해 Telegram 메시지를 보내고 받으려면 Telegram 봇이 필요합니다. 봇은 공식 BotFather를 통해 생성하며 무료입니다.

1단계 — Telegram 봇 생성

  1. Telegram을 열고 @BotFather를 검색합니다.
  2. BotFather와 채팅을 시작하고 /newbot 명령을 보냅니다.
  3. 봇의 이름을 선택합니다 (예: Klacks Notifications).
  4. 봇의 사용자명을 선택합니다 (bot으로 끝나야 함, 예: klacks_notify_bot).
  5. BotFather가 봇 토큰을 제공합니다. 이 토큰을 복사하세요 — 다음 단계에서 필요합니다.

참고: 봇 토큰은 비밀번호처럼 취급하세요. 공개적으로 공유하지 말고 안전하게 보관하세요.

2단계 — Klacks에서 봇 구성

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. 추가를 클릭합니다.
  3. 다음과 같이 필드를 입력합니다:
필드설명예
이름고유한 내부 이름telegram-bot
표시 이름사용자 인터페이스에 표시됨Telegram
프로바이더 유형Telegram 선택Telegram
봇 토큰BotFather에서 받은 토큰123456:ABC-DEF1234ghIkl-zyx57W2v...
Webhook URL수신 메시지용 URL (4단계 참조)https://example.com/api/messaging/webhook/telegram
활성화프로바이더 활성화/비활성화예
  1. 저장을 클릭합니다.
  2. 프로바이더 목록의 테스트 버튼으로 연결을 테스트합니다.

3단계 — 챗봇(Klacksy)을 통한 메시지 전송

Telegram 봇이 구성되고 활성화되면 챗봇 Klacksy가 메시지를 보낼 수 있습니다. 채팅 ID를 알 필요 없이 직원 또는 고객의 이름만 사용하면 됩니다: 

"홍길동에게 메시지 보내기: 내일 오후 2시에 예약이 있습니다."
"김영희에게 메시지 보내기: 근무가 교대되었습니다."
"이철수에게 월요일 근무가 취소되었다고 알려주세요."

Klacksy가 자동으로 수신자의 Telegram 사용자명을 조회하고 구성된 봇을 사용합니다. 이 과정은 사용자에게 완전히 투명합니다 — 이름으로 작업하면 Klacksy가 나머지를 처리합니다.

전제 조건: 수신자의 Telegram 사용자명이 Klacks에 저장되어 있어야 합니다 (주소 → 연락처 정보).

4단계 — 수신 메시지 (Webhook)

수신 Telegram 메시지를 Klacks에 표시하려면 Webhook을 구성해야 합니다.

  1. Klacks 인스턴스가 HTTPS를 통해 공개적으로 접근 가능한지 확인합니다.
  2. 프로바이더 설정에서 다음 Webhook URL을 입력합니다: 
    https://{your-domain}/api/messaging/webhook/telegram
  3. 프로바이더가 저장되고 활성화되면 Klacks가 자동으로 Telegram에 Webhook을 등록합니다.
  4. 수신 메시지가 이제 메시징 수신함에 실시간으로 표시됩니다.

문제 해결

문제가능한 원인해결 방법
연결 테스트 실패 봇 토큰이 유효하지 않음 BotFather에서 /mybots로 토큰을 확인하세요
메시지가 전달되지 않음 수신자가 봇을 시작하지 않음 수신자가 Telegram에서 봇을 열고 /start를 보내야 합니다
수신자를 찾을 수 없음 Telegram 사용자명이 저장되지 않음 주소 → 연락처 정보에서 Telegram 사용자명을 입력하세요
수신 메시지 누락 Webhook URL에 접근할 수 없음 도메인이 HTTPS를 통해 접근 가능하고 URL이 올바른지 확인하세요
Webhook 등록 실패 유효한 SSL 인증서 없음 Telegram은 유효한 SSL 인증서를 요구합니다 (자체 서명 불가)
