Microsoft Teams 설정

사전 준비

Klacks를 통해 Microsoft Teams로 메시지를 보내고 받으려면 활성 Teams 라이선스가 있는 Microsoft 365 Business 계정이 필요합니다. 통합은 Azure 앱 등록과 Azure Bot Service를 통해 수행됩니다.

참고: Azure Bot Service는 표준 채널(Teams 포함)에 대해 무료입니다. 봇 서비스에 대한 추가 비용은 없습니다. 단, Microsoft 365 Business 라이선스가 필요합니다.

1단계 — Azure 앱 등록 및 봇 생성

  1. Azure Portal (portal.azure.com)에 로그인합니다.
  2. 앱 등록으로 이동하고 새 등록을 클릭합니다.
  3. 이름을 입력하고 (예: Klacks-Teams-Bot) 적절한 계정 유형을 선택합니다.
  4. 애플리케이션 (클라이언트) ID를 확인합니다 — 이것이 클라이언트 ID입니다.
  5. 디렉토리 (테넌트) ID를 확인합니다 — 이것이 테넌트 ID입니다.
  6. 인증서 & 비밀에서 새 클라이언트 시크릿을 생성하고 값을 확인합니다.
  7. Azure Bot (이전 Bot Channel Registration)으로 이동하여 새 봇을 생성합니다.
  8. 봇을 생성한 앱 등록에 연결합니다.
  9. 채널에서 Microsoft Teams 채널을 활성화합니다.

2단계 — Klacks에서 Microsoft Teams 구성

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. 추가를 클릭합니다.
  3. 다음과 같이 필드를 입력합니다:
필드설명예
이름고유한 내부 이름teams-main
표시 이름사용자 인터페이스에 표시됨Microsoft Teams
프로바이더 유형Teams 선택Teams
테넌트 IDAzure Portal의 디렉토리 (테넌트) IDa1b2c3d4-e5f6-7890-abcd-ef1234567890
클라이언트 ID앱 등록의 애플리케이션 (클라이언트) ID12345678-abcd-efgh-ijkl-1234567890ab
클라이언트 시크릿인증서 & 비밀의 시크릿 값abc~DEF123...
활성화프로바이더 활성화/비활성화예
  1. 저장을 클릭합니다.
  2. 프로바이더 목록의 테스트 버튼으로 연결을 테스트합니다.

3단계 — 챗봇(Klacksy)을 통한 메시지 전송

Teams 프로바이더가 구성되고 활성화되면 챗봇 Klacksy가 Microsoft Teams를 통해 메시지를 보낼 수 있습니다. 직원 또는 고객의 이름만 사용하면 됩니다: 

"홍길동에게 메시지 보내기: 내일 오후 2시에 예약이 있습니다."
"김영희에게 메시지 보내기: 근무가 교대되었습니다."
"이철수에게 월요일 근무가 취소되었다고 알려주세요."

Klacksy가 자동으로 연결된 Teams 사용자를 조회하고 Microsoft Teams를 통해 메시지를 보냅니다. 이 과정은 사용자에게 완전히 투명합니다 — 이름으로 작업하면 Klacksy가 나머지를 처리합니다.

전제 조건: Teams 사용자가 해당 Klacks 연락처에 연결되어 있어야 합니다 (주소 → 연락처 정보).

4단계 — 수신 메시지 (Webhook)

수신 Teams 메시지를 Klacks에 표시하려면 Webhook을 구성해야 합니다.

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. Teams 프로바이더의 자동 생성된 Webhook 시크릿을 확인합니다.
  3. Azure Portal의 봇에서 메시징 엔드포인트 URL을 구성합니다: 
    https://{your-domain}/api/messaging/webhook/teams
  4. 수신 메시지가 이제 메시징 수신함에 실시간으로 표시됩니다.

문제 해결

문제가능한 원인해결 방법
연결 테스트 실패 클라이언트 ID, 테넌트 ID 또는 클라이언트 시크릿이 잘못됨 Azure Portal에서 자격 증명을 확인하세요
메시지가 전달되지 않음 봇에서 Teams 채널이 활성화되지 않음 Azure Bot 구성에서 Teams 채널을 활성화하세요
수신자가 메시지를 받지 못함 Teams 사용자가 Klacks 연락처에 연결되지 않음 주소 → 연락처 정보에서 Teams 사용자를 연결하세요
수신 메시지 누락 Webhook URL이 구성되지 않음 Azure Portal에서 메시징 엔드포인트 URL을 확인하세요
인증 오류 (401/403) 클라이언트 시크릿 만료 Azure Portal에서 새 클라이언트 시크릿을 생성하고 Klacks에서 업데이트하세요
