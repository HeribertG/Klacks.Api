Signal 메시징 설정

사전 준비

Klacks를 통해 Signal 메시지를 보내고 받으려면 실행 중인 Signal-CLI REST API 인스턴스가 필요합니다. 이것은 REST 인터페이스를 통해 Signal에 접근할 수 있게 하는 오픈소스 프로젝트입니다.

  • 서버에 Docker가 설치되어 있어야 함
  • Signal에 등록할 전화번호 (전용 번호 권장)
  • Docker 이미지: bbernhard/signal-cli-rest-api

1단계 — Signal-CLI REST API 설정

  1. Docker로 Signal-CLI 컨테이너를 시작합니다: 
    docker run -d --name signal-cli-rest-api \
  --restart=unless-stopped \
  -p 8080:8080 \
  -v signal-cli-config:/home/.local/share/signal-cli \
  -e MODE=normal \
  bbernhard/signal-cli-rest-api
  2. API를 통해 전화번호를 등록합니다: 
    curl -X POST 'http://localhost:8080/v1/register/{number}' \
  -H 'Content-Type: application/json' \
  -d '{"captcha": "", "use_voice": false}'

    {number}를 +82010... 형식의 번호로 대체하세요

  3. 수신된 코드로 번호를 인증합니다: 
    curl -X POST 'http://localhost:8080/v1/register/{number}/verify/{code}'
  4. 헬스 체크로 API를 테스트합니다: 
    curl http://localhost:8080/v1/about

참고: 자세한 구성 정보는 Signal-CLI REST API 문서를 참조하세요.

2단계 — Klacks에서 Signal 구성

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. 추가를 클릭합니다.
  3. 다음과 같이 필드를 입력합니다:
필드설명예
이름고유한 내부 이름signal-main
표시 이름사용자 인터페이스에 표시됨Signal
프로바이더 유형Signal 선택Signal
Signal 번호등록된 전화번호 (국제 형식)+821012345678
API URLSignal-CLI REST API의 URLhttp://localhost:8080
활성화프로바이더 활성화/비활성화예
  1. 저장을 클릭합니다.
  2. 프로바이더 목록의 테스트 버튼으로 연결을 테스트합니다.

3단계 — 챗봇(Klacksy)을 통한 메시지 전송

Signal 프로바이더가 구성되고 활성화되면 챗봇 Klacksy가 메시지를 보낼 수 있습니다. 전화번호를 알 필요 없이 직원 또는 고객의 이름만 사용하면 됩니다: 

"홍길동에게 메시지 보내기: 내일 오후 2시에 예약이 있습니다."
"김영희에게 Signal 메시지 보내기: 근무가 교대되었습니다."
"이철수에게 월요일 근무가 취소되었다고 알려주세요."

Klacksy가 자동으로 수신자의 전화번호를 조회하고 구성된 Signal 프로바이더를 사용합니다. 이 과정은 사용자에게 완전히 투명합니다 — 이름으로 작업하면 Klacksy가 나머지를 처리합니다.

전제 조건: 수신자의 휴대전화 번호가 Klacks에 저장되어 있어야 합니다 (주소 → 연락처 정보).

4단계 — 수신 메시지 (Webhook)

수신 Signal 메시지를 Klacks에 표시하려면 Webhook을 구성해야 합니다.

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. Signal 프로바이더의 자동 생성된 Webhook 시크릿을 확인합니다.
  3. Signal-CLI REST API에서 Webhook URL을 구성합니다: 
    https://{your-domain}/api/messaging/webhook/signal
  4. 수신 메시지가 이제 메시징 수신함에 실시간으로 표시됩니다.

문제 해결

문제가능한 원인해결 방법
연결 테스트 실패 Signal-CLI REST API에 접근할 수 없음 Docker 컨테이너가 실행 중인지 확인하세요: docker ps
메시지가 전달되지 않음 Signal 번호가 등록되지 않았거나 인증되지 않음 API를 통해 등록 및 인증을 다시 수행하세요
수신자가 메시지를 받지 못함 수신자가 Signal을 사용하지 않거나 번호가 잘못됨 수신자가 Signal을 설치했고 번호가 +{국가코드}{번호} 형식으로 저장되어 있는지 확인하세요
수신 메시지 누락 Webhook이 구성되지 않음 Signal-CLI REST API 구성에서 Webhook URL을 확인하세요
컨테이너가 시작되지 않음 포트 8080이 이미 사용 중 다른 포트를 사용하세요: -p 9090:8080 그리고 Klacks에서 API URL을 그에 맞게 업데이트하세요
