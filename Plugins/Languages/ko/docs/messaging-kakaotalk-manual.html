카카오톡 설정

사전 준비

Klacks를 통해 카카오톡으로 메시지를 보내고 받으려면 Kakao Developers 계정과 등록된 카카오톡 채널이 필요합니다. 카카오톡은 한국에서 약 93%의 시장 점유율을 가진 대표적인 메신저입니다.

참고: 비즈니스 메시지에는 카카오톡 채널이 필요합니다. 채널 없이 개인 메시지는 보낼 수 없습니다.

1단계 — Kakao Developers 계정 생성

  1. developers.kakao.com에서 가입합니다.
  2. 개발자 대시보드에서 새 애플리케이션(앱)을 생성합니다.
  3. 앱 설정에서 앱 키를 확인합니다.
  4. 카카오톡 채널을 등록하고 앱에 연결합니다.
  5. 메시지 템플릿을 생성하고 템플릿 ID를 확인합니다.

2단계 — Klacks에서 카카오톡 구성

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. 추가를 클릭합니다.
  3. 다음과 같이 필드를 입력합니다:
필드설명예
이름고유한 내부 이름kakaotalk-main
표시 이름사용자 인터페이스에 표시됨카카오톡
프로바이더 유형KakaoTalk 선택KakaoTalk
앱 키Kakao Developers 대시보드의 앱 키abc123def456...
템플릿 ID메시지 템플릿의 ID12345
활성화프로바이더 활성화/비활성화예
  1. 저장을 클릭합니다.
  2. 프로바이더 목록의 테스트 버튼으로 연결을 테스트합니다.

3단계 — 챗봇(Klacksy)을 통한 메시지 전송

카카오톡 프로바이더가 구성되고 활성화되면 챗봇 Klacksy가 메시지를 보낼 수 있습니다. 전화번호를 알 필요 없이 직원 또는 고객의 이름만 사용하면 됩니다: 

"홍길동에게 메시지 보내기: 내일 오후 2시에 예약이 있습니다."
"김영희에게 메시지 보내기: 근무가 교대되었습니다."
"이철수에게 월요일 근무가 취소되었다고 알려주세요."

Klacksy가 자동으로 수신자의 전화번호를 조회하고 구성된 카카오톡 프로바이더를 사용합니다. 이 과정은 사용자에게 완전히 투명합니다 — 이름으로 작업하면 Klacksy가 나머지를 처리합니다.

전제 조건: 수신자의 휴대전화 번호가 Klacks에 저장되어 있어야 합니다 (주소 → 연락처 정보).

4단계 — 수신 메시지 (Webhook)

수신 카카오톡 메시지를 Klacks에 표시하려면 Webhook을 구성해야 합니다.

  1. 설정 → 메시징 프로바이더로 이동합니다.
  2. 카카오톡 프로바이더의 자동 생성된 Webhook 시크릿을 확인합니다.
  3. Kakao Developers 대시보드에서 Webhook URL을 구성합니다: 
    https://{your-domain}/api/messaging/webhook/kakaotalk
  4. 수신 메시지가 이제 메시징 수신함에 실시간으로 표시됩니다.

문제 해결

문제가능한 원인해결 방법
연결 테스트 실패 앱 키 또는 템플릿 ID가 잘못됨 Kakao Developers 대시보드에서 자격 증명을 확인하세요
메시지가 전달되지 않음 카카오톡 채널이 연결되지 않음 채널이 앱에 연결되어 있는지 확인하세요
수신자가 메시지를 받지 못함 휴대전화 번호가 등록되지 않았거나 수신자가 카카오톡 사용자가 아님 수신자가 카카오톡을 사용하고 있고 번호가 올바른지 확인하세요
수신 메시지 누락 Webhook이 구성되지 않음 Kakao Developers 대시보드에서 Webhook URL을 확인하세요
템플릿 오류 템플릿이 승인되지 않았거나 ID가 잘못됨 Kakao Developers 대시보드에서 템플릿 상태를 확인하세요
