身份提供商手册

简介

通过使用身份提供商，您可以通过LDAP、Active Directory、OAuth2/OpenID Connect等 外部目录服务进行用户身份验证。

支持的提供商类型：

  • LDAP - 标准轻量级目录访问协议
  • Active Directory - Microsoft Active Directory
  • OAuth2 - OAuth 2.0 授权框架
  • OpenID Connect - 基于OAuth 2.0的身份层

LDAP与Active Directory的区别

属性LDAP（OpenLDAP）Active Directory
用户属性uidsAMAccountName
地址属性streetstreetAddress
称谓属性-personalTitle
国家属性-c
认证方式简单绑定Negotiate/Kerberos

典型用户过滤器

LDAP:             (objectClass=person)
                  (objectClass=inetOrgPerson)

Active Directory: (&(objectClass=user)(objectCategory=person))
                  (&(objectClass=user)(!(userAccountControl:1.2.840.113556.1.4.803:=2)))

客户导入

启用"用于客户导入"后，将导入以下数据：

  • 姓名 - sn（姓氏）或 displayName
  • 名字 - givenName
  • 职位 - title（职位名称）
  • 公司 - company
  • 地址 - street/streetAddress、postalCode、l（城市）、st（省/州）、c（国家）
  • 性别 - Active Directory情况下，从personalTitle（Herr/Frau）获取

注意：使用OpenLDAP时，性别默认为"中性"，可手动修正。

提示

使用Synology NAS作为LDAP服务器

您可以使用Synology NAS作为LDAP服务器。在这种情况下，需要安装"LDAP Server"软件包 或使用基于OpenLDAP的Docker容器。

  • 提供商类型："Active Directory"（用于回退认证）
  • 用户过滤器：(objectClass=inetOrgPerson)
  • 用户DN格式：cn=username,ou=users,dc=example,dc=com

测试连接

请使用"测试连接"按钮来验证设置。 连接成功后，将显示找到的用户数量和一些示例用户名。
