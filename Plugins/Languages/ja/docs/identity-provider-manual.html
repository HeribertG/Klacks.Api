IDプロバイダー マニュアル

はじめに

IDプロバイダーを使用することで、LDAP、Active Directory、OAuth2/OpenID Connectなどの 外部ディレクトリサービスを通じてユーザー認証を行うことができます。

対応プロバイダーの種類：

  • LDAP - 標準軽量ディレクトリアクセスプロトコル
  • Active Directory - Microsoft Active Directory
  • OAuth2 - OAuth 2.0 認可フレームワーク
  • OpenID Connect - OAuth 2.0 上のIDレイヤー

クライアントインポート

「クライアントインポートに使用」を有効にした場合、以下のデータがインポートされます：

  • 氏名 - sn（姓）または displayName
  • 名前 - givenName
  • 役職 - title（役職名）
  • 会社 - company
  • 住所 - street/streetAddress、postalCode、l（市区町村）、st（都道府県）、c（国）
  • 性別 - Active Directoryの場合、personalTitle（Herr/Frau）から取得

注意：OpenLDAPの場合、性別はデフォルトで「ニュートラル」となり、手動で修正できます。

ヒント

Synology NAS を LDAPサーバーとして使用

Synology NASをLDAPサーバーとして使用できます。その場合、「LDAPサーバー」パッケージ またはOpenLDAPを使用したDockerコンテナーをインストールする必要があります。

  • プロバイダータイプ：「Active Directory」（フォールバック認証用）
  • ユーザーフィルター：(objectClass=inetOrgPerson)
  • ユーザーDN形式：cn=username,ou=users,dc=example,dc=com

接続のテスト

「接続をテスト」ボタンを使用して設定を確認してください。 接続が成功すると、見つかったユーザー数といくつかのサンプルユーザー名が表示されます。

トラブルシューティング

問題解決策
接続に失敗ホスト、ポート、SSL設定を確認してください
バインドに失敗バインドDNとパスワードを確認してください
ユーザーが見つからないベースDNとユーザーフィルターを確認してください
住所がインポートされないOpenLDAPは「streetAddress」ではなく「street」を使用します
