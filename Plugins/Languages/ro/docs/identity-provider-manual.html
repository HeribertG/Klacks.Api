Manual Furnizor de Identitate

Introducere

Furnizorii de identitate permit autentificarea utilizatorilor prin servicii de directoare externe precum LDAP, Active Directory, OAuth2/OpenID Connect.

Tipuri de furnizori acceptate:

  • LDAP - Protocol standard de acces la directoare
  • Active Directory - Microsoft Active Directory
  • OAuth2 - Cadru de autorizare OAuth 2.0
  • OpenID Connect - Strat de identitate peste OAuth 2.0

Import clienți

Când „Utilizare pentru import clienți" este activat, următoarele date sunt importate:

  • Nume - sn (nume de familie) sau displayName
  • Prenume - givenName
  • Titlu - title (funcție)
  • Companie - company
  • Adresă - street/streetAddress, postalCode, l (localitate), st (județ), c (țară)
  • Gen - Pentru Active Directory, din personalTitle (Herr/Frau)

Notă: Pentru OpenLDAP, genul implicit este „Neutru" și poate fi corectat manual.

Sfaturi

Utilizarea Synology NAS ca server LDAP

Puteți utiliza un Synology NAS ca server LDAP. În acest caz, trebuie să instalați pachetul „LDAP Server" sau un container Docker cu OpenLDAP.

  • Tip furnizor: „Active Directory" (pentru autentificare de rezervă)
  • Filtru utilizatori: (objectClass=inetOrgPerson)
  • Format DN utilizator: cn=username,ou=users,dc=example,dc=com

Testarea conexiunii

Utilizați butonul „Testare conexiune" pentru a verifica configurarea. La o conexiune reușită, se afișează numărul de utilizatori găsiți și câteva exemple de nume de utilizatori.
