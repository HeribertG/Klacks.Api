Manual Pembekal ID

Pengenalan

Dengan menggunakan Pembekal ID, anda boleh melaksanakan pengesahan pengguna melalui perkhidmatan direktori luaran seperti LDAP, Active Directory, OAuth2/OpenID Connect.

Jenis pembekal yang disokong:

  • LDAP - Protokol Akses Direktori Ringan Standard
  • Active Directory - Microsoft Active Directory
  • OAuth2 - Rangka Kerja Kebenaran OAuth 2.0
  • OpenID Connect - Lapisan Identiti di atas OAuth 2.0

Import Klien

Jika "Guna untuk import klien" diaktifkan, data berikut akan diimport:

  • Nama keluarga - sn (nama keluarga) atau displayName
  • Nama pertama - givenName
  • Jawatan - title (jawatan)
  • Syarikat - company
  • Alamat - street/streetAddress, postalCode, l (bandar), st (negeri), c (negara)
  • Jantina - Untuk Active Directory, diambil dari personalTitle (Herr/Frau)

Nota: Untuk OpenLDAP, jantina lalai ialah "Neutral" dan boleh diperbetulkan secara manual.

Petua

Menggunakan Synology NAS sebagai Pelayan LDAP

Anda boleh menggunakan Synology NAS sebagai pelayan LDAP. Dalam kes ini, anda perlu memasang pakej "Pelayan LDAP" atau kontena Docker dengan OpenLDAP.

  • Jenis pembekal: "Active Directory" (untuk pengesahan fallback)
  • Penapis pengguna: (objectClass=inetOrgPerson)
  • Format DN pengguna: cn=username,ou=users,dc=example,dc=com

Menguji Sambungan

Gunakan butang "Uji Sambungan" untuk mengesahkan tetapan anda. Jika sambungan berjaya, bilangan pengguna yang ditemui dan beberapa contoh nama pengguna akan dipaparkan.
